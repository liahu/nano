#项目概述
    本项目是一个Golang后端服务，为智能小机器人设备和手机App提供连接和管理服务。

    核心业务：
    - App与机器人设备先通过蓝牙本地配对，配对成功后在服务端建立账号-设备绑定关系
    - 一个App账号最多可绑定和管理10个机器人设备
    - App通过HTTP协议查看和管理已绑定的机器人
    - 机器人设备通过HTTP进行设备注册、状态上报、OTA升级等操作，通过WebSocket进行实时语音识别等双向通信

    服务职责：
    - 提供App端HTTP接口：账号管理、设备绑定/解绑、设备控制、OTA升级等
    - 提供机器人端HTTP接口：设备注册、状态上报、OTA升级等
    - 提供机器人端WebSocket接口：实时语音识别等双向通信场景


#HTTP接口：
    App端接口：
        账号管理：
            账号注册
                Method: POST
                URL: /api/v1/users/register
                Body: {"username": "string", "phone": "string", "password": "string", "wechat": "string"}
                响应：{"token": "string"}（注册成功即登录成功） jwt
            账号登录
                Method: POST
                URL: /api/v1/users/login
                Body: {"phone": "string", "password": "string"}
                响应：{"token": "string"} jwt
            账号信息查询
                Method: GET
                URL: /api/v1/users/info
                说明：需要携带用户Token
            账号信息修改
                Method: POST
                URL: /api/v1/users/info
                Body: {"username": "string", "password": "string", "wechat": "string"}
                说明：需要携带用户Token
        
        设备管理：
            设备绑定（建立账号-设备关联）
                Method: POST
                URL: /api/v1/devices/bind
                Body: {"device_id": "string"}
                说明：需要携带用户Token
            设备解绑（解除账号-设备关联）
                Method: POST
                URL: /api/v1/devices/unbind
                Body: {"device_id": "string"}
                说明：需要携带用户Token
            绑定设备列表
                Method: GET
                URL: /api/v1/devices
                说明：需要携带用户Token
            设备详情
                Method: GET
                URL: /api/v1/devices/{device_id}
                说明：需要携带用户Token
            设备控制
                Method: POST
                URL: /api/v1/devices/{device_id}/control
                Body: {"command": "string", "params": {"key": "value"}}
                说明：需要携带用户Token
            设备信息修改
                Method: POST
                URL: /api/v1/devices/{device_id}
                Body: {"name": "string"}
                说明：需要携带用户Token
        
        OTA管理：
            最新OTA版本查询
                Method: GET
                URL: /api/v1/ota/latest
                说明：需要携带用户Token
            设备OTA版本查询
                Method: GET
                URL: /api/v1/devices/{device_id}/ota
                说明：需要携带用户Token
            设备OTA升级
                Method: POST
                URL: /api/v1/devices/{device_id}/ota/upgrade
                Body: {"version": "string"}
                说明：需要携带用户Token
    
    机器人端接口：
        设备注册
            Method: POST
            URL: /api/v1/devices/register
            Body: {"hardware_id": "string", "version": "string"}
            响应：{"device_id": "string", "secret": "string"}
        设备状态上报
            Method: POST
            URL: /api/v1/devices/status
            Body: {"status": "string", "battery": number, "version": "string"}
            说明：需要携带Device-ID和Signature
        OTA版本查询 （可能不需要）
            Method: GET
            URL: /api/v1/ota/check
            说明：需要携带Device-ID和Signature
        OTA版本下载
            Method: GET
            URL: /api/v1/ota/download/{version}
            说明：需要携带Device-ID和Signature
        升级状态上报
            Method: POST
            URL: /api/v1/devices/ota/status
            Body: {"status": "string", "progress": number, "error": "string"}
            说明：需要携带Device-ID和Signature
    
    Admin后台接口：
        用户管理：
            用户信息查询
                Method: GET
                URL: /api/v1/admin/users
                说明：需要Admin Token
            用户管理（增删改）
                Method: POST
                URL: /api/v1/admin/users
                Body: {"action": "create|update|delete", "user": {"id": "string", "username": "string", "phone": "string"}}
                说明：需要Admin Token
        设备管理：
            设备信息查询
                Method: GET
                URL: /api/v1/admin/devices
                说明：需要Admin Token
            设备管理（增删改）
                Method: POST
                URL: /api/v1/admin/devices
                Body: {"action": "create|update|delete", "device": {"id": "string", "version": "string", "user_id": "string"}}
                说明：需要Admin Token
        OTA管理：
            OTA版本上传
                Method: POST
                URL: /api/v1/admin/ota/upload
                Body: multipart/form-data (包含bin文件和版本信息)
                说明：需要Admin Token
            OTA版本管理
                Method: POST
                URL: /api/v1/admin/ota
                Body: {"action": "publish|unpublish", "version": "string"}
                说明：需要Admin Token
            OTA版本列表
                Method: GET
                URL: /api/v1/admin/ota
                说明：需要Admin Token


#websocket接口：
    机器人端发起：
        建立连接
        发送语音

    服务端发起：
        推送控制指令（包含动作控制和OTA）
        推动语音流的回复
        



#项目技术要求
    框架： gin-gonic/gin

    日志：log/slog
        日志需要有一个traceID，便于跟踪，traceID的生成，要和用户id有关系。
        实现方式：封装mlog包，提供统一的日志接口，支持文件和控制台输出，支持日志轮转。

    项目结构：
        src/
        ├── api/             # API接口层
        │   ├── handlers/    # 请求处理器
        │   ├── middlewares/ # 中间件
        │   └── routes/      # 路由定义
        ├── services/        # 业务逻辑层
        ├── models/          # 数据模型
        ├── db/              # 数据库操作
        │   ├── db.go        # 数据库连接管理
        │   └── repositories/ # 数据访问层（Repository模式）
        ├── configs/         # 配置管理
        ├── utils/           # 工具函数
        │   ├── mlog/        # 日志系统
        │   └── log.go       # 日志接口
        └── main.go          # 应用入口



#数据库要求：
    db:sqlite

    链接方式：gorm.io/gorm

    设计模式：Repository模式
        实现方式：db包提供NewDB()函数创建数据库连接，repositories包通过依赖注入方式接受db对象，实现数据访问逻辑。
        优势：避免导入循环，支持依赖注入，便于测试和扩展。

    table：
        user：
            id (primary key, varchar(36))
            username (varchar(50), not null)
            phone (varchar(20), unique, not null)
            wechat (varchar(50))
            password (varchar(100), not null)
            created_at (datetime)
            updated_at (datetime)

        device：
            id (primary key, varchar(36))
            version (varchar(20), not null)
            status (varchar(20), not null, default: 'inactive')
            user_id (varchar(36), index)
            secret (varchar(100), not null)
            created_at (datetime)
            updated_at (datetime)


#配置：
    本地yaml文件，文件名为config.yaml，主要配置服务的地址，等信息。
